## 3. Tech Stack

**Pure static:** HTML + CSS + Vanilla JS. Zero frameworks. Zero bundlers. Zero runtime fetches.

**Fonts:**
- Syne (headings, 800/700/600) — Google Fonts CDN
- DM Sans (body, 300/400/500) — Google Fonts CDN
- DM Mono (badges, mono — premium tier) — Google Fonts CDN
- Space Mono (badges, mono — free tier) — Google Fonts CDN

**Icons:** Font Awesome 6.0.0 — cdnjs CDN
**PDF (free tier):** jsPDF 2.5.1 — cdnjs CDN
**PDF (premium tier):** `window.print()` with `@media print` stylesheet — no library needed

**Framework (planned):** Svelte — compiles to vanilla JS, zero runtime overhead, consistent with static-first doctrine.

## 4. Architecture Principles

- HTML routes. JS behaves. CSS styles. Python builds. Never cross these lines.
- Questions are embedded at build time as `const QUESTIONS = [...]` — zero runtime loading
- `<a href="">` is the only routing mechanism — JS never navigates between pages
- **Exception:** Session gate redirect in premium pages uses `window.location.href` — single sanctioned use only
- No page dead-ends on GitHub's 404 — all dead routes go to `coming-soon.html`
- See `ANTIFAIL_DOCTRINE_V2.md` for the full rules

## 6. Build Pipeline

### Free Quiz Tier (existing)
1. Write/edit questions in `build.py` under `RAW_QUESTIONS`
2. Run `python build.py` locally
3. Files write to `./output/`
4. Manually copy output files to repo root and push
5. GitHub Pages redeploys automatically

### Premium Papers (new)
1. Source questions from Ada file (`STU002_Sunny-Ada_Chemistry.html`) and/or MEPC file (`MEPC_Chemistry150_Principles.html`)
2. Run Python selection script: topic allocation → deduplication check → option shuffle → answer key balance
3. Verify: 50 questions, A=13/B=13/C=12/D=12, zero cross-paper overlap, `node --check` passes
4. Stamp questions into Paper 1 template (3 string replacements: title tag, header span, QUESTIONS block)
5. Activate card in hub HTML (4 attribute changes per card)
6. Deploy all changed files to repo root

## 8. Premium System Details

### Auth Architecture
- Single password input → SHA-256 (`crypto.subtle`, HTTPS only) → hardcoded hash registry
- Password encodes both student identity and subject (e.g. `kon-che-XXXXXX` → Chemistry, `kon-eng-XXXXXX` → English)
- On match: writes session to `sessionStorage` → redirects to correct hub
- Session TTL: 3 hours. Dies with tab.
- 8 students: Conve, Kaosara, Habibat, Oyin, Dorcas, Famous, Obasesam, Prosper

### Session Schema
```js
{ name: "StudentName", subject: "chemistry" | "english", expires: timestamp }
```
Key: `ga_premium_session`

### Hub Gate
Each hub checks `session.subject === 'chemistry'` (or 'english') on load. Mismatch or missing → redirect to auth. This is the only sanctioned use of `window.location.href`.

### Watermark
Student name injected from session into a fixed-position div. Diagonal tiled text, `opacity: 0.022`, pointer-events none, `user-select: none`. Appears on hub and all quiz pages.

### Quiz Paper Invariants (enforce on every paper)
| Invariant | Value |
|---|---|
| Questions per paper | Exactly 50 |
| Answer distribution | A=13, B=13, C=12, D=12 |
| Cross-paper overlap | 0 |
| Question IDs | 1–50, reset per paper |
| Timer | 1800s (30 min) |
| JS dependencies | None (fully self-contained) |
| PDF | `window.print()` + `@media print`, student name stamped |
| Gate | Session only — no password on quiz page |

## 11. Student Feedback Loop

Students download PDF score reports after each quiz. They DM these PDFs to the owner. The owner reviews wrong answers, checks question quality (answer key, wording, explanation), and fixes in `build.py` (free tier) or flags for next build cycle (premium).

The owner answers every question personally before it goes live. The owner is the first student.
